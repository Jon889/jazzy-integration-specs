Configuration

public struct Configuration

Options which control the behavior of a Resource.

See also

Service.configure(...)

General Resource Behavior
  • expirationTime

    Time before valid data is considered stale by Resource.loadIfNeeded().

    The default is 30 seconds.

    Note

    This property is configured at the resource level, and does not depend on the HTTP method of any request. Siesta uses the value configured for GET; if you override this for other HTTP methods, Siesta will ignore it.

    Declaration

    Swift

    public var expirationTime: NSTimeInterval = 30
  • retryTime

    Time Resource.loadIfNeeded() will wait before allowing a retry after a failed request.

    The default is 1 second.

    Note

    This property is configured at the resource level, and does not depend on the HTTP method of any request. Siesta uses the value configured for GET; if you override this for other HTTP methods, Siesta will ignore it.

    Declaration

    Swift

    public var retryTime: NSTimeInterval = 1
  • persistentCache

    An optional store to maintain the state of resources between app launches.

    Note

    This property is configured at the resource level, and does not depend on the HTTP method of any request. Siesta uses the value configured for GET; if you override this for other HTTP methods, Siesta will ignore it.

    Declaration

    Swift

    public var persistentCache: EntityCache? = nil

Request Handling
  • headers

    Default request headers.

    Note

    Resource.request(...) accepts a requestMutation closure than can change the HTTP method of a request. If you override configuration based on HTTP method, then unlike other configuration properties, headers depends on the initially requested, pre-mutation method of a request. All other configuration properties will depend on the post-mutation request method.

    Declaration

    Swift

    public var headers: [String:String] = [:]
  • beforeStartingRequest(_:)

    Adds a closure to be called after a Request is created, but before it is started. Use this to add response hooks or cancel the request before sending.

    Declaration

    Swift

    public mutating func beforeStartingRequest(callback: (Resource, Request) -> Void)
  • responseTransformers

    A sequence of parsers to be applied to responses.

    You can add custom parsing using:

    $0.config.responseTransformers.add(MyCustomTransformer())
$0.config.responseTransformers.add(MyCustomTransformer(), contentTypes: ["foo/bar"])

    By default, the transformer sequence includes JSON, image, and plain text parsing. You can remove this default behavior by clearing the sequence:

    $0.config.responseTransformers.clear()

    See also

    Service.configureTransformer(...)

    Declaration

    Swift

    public var responseTransformers: TransformerSequence = TransformerSequence()
  • progressReportingInterval

    Interval at which request hooks & observers receive progress updates. This affects how frequently Request.onProgress(_:) and ResourceObserver.resourceRequestProgress(_:progress:) are called, and how often the Request.progress property (which is partially time-based) is updated.

    Declaration

    Swift

    public var progressReportingInterval: NSTimeInterval = 0.05

Creating Configurations
  • Builder

    Holds a mutable configuration while closures passed to Service.configure(...) modify it.

    The reason that method doesn’t just accept a closure with an inout param is that doing so requires a messy flavor of closure declaration that makes the API much harder to use:

    configure("/things/​*") { (inout config: Configuration) in config.retryTime = 1 }

    This wrapper class allows usage to instead look like:

    configure("/things/​*") { $0.config.retryTime = 1 }
    See more
